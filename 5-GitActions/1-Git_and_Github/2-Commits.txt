We would discuss git commits here

Commits are like snapshots of code at a particular point in time

git add

    This is used to group files that would ready for snapshot later,
    it like bring people together that you would be ready to snap later.


    git add file1 file2

    git add .    # this would add all


git commit

    This is the snapping of the added files, add is the staging while committing them
    is the snapshot in time.

        git commit -m "message"
